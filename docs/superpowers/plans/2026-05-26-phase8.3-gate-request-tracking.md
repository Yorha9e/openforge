# Phase 8.3: 待审批门禁请求持久化 (Gate Request Tracking) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 完成挂起审批状态表 `gate_request` 的持久化、自动超时流、认领机制及状态跟踪，保证审批挂起过程（Await-Review）在实例外和持久连接外均能被完全追踪和自愈。

**Architecture:** 
1. 编写 `GateRequestRepository` 管理挂起审批记录（`pending` / `approved` / `rejected` / `timeout`）。
2. 当 L3/L4 Pipeline 遇到人工门禁时，在后台异步创建一条带有 5 分钟 Expires 时效的 `gate_request`。
3. 用户在 UI 的 Review Inbox 认领此项（向 `gate_request` 写入 `claimed_by`），并在审批通过后更新其状态。
4. 后台 Cron 扫描超时、对超时的 pending 进行 `timeout` 处理并促使 Pipeline 异常退出，防止线程或协程无限期僵死挂起。

**Tech Stack:** Go, PostgreSQL, Cron Worker

- [ ] **Step 2: 在 PostgreSQL 中实现 `pg_gate_request_repository.go`**

```go
package adapter

import (
	"context"
	"database/sql"
	"openforge/internal/pipeline/port"
)

type PGGateRequestRepository struct {
	db *sql.DB
}

func NewPGGateRequestRepository(db *sql.DB) *PGGateRequestRepository {
	return &PGGateRequestRepository{db: db}
}

func (r *PGGateRequestRepository) CreateRequest(ctx context.Context, req *port.DBGateRequest) error {
	_, err := r.db.ExecContext(ctx, `
		INSERT INTO gate_request (pipeline_id, stage, status, requested_by, timeout_at)
		VALUES ($1, $2, $3, $4, $5)
	`, req.PipelineID, req.Stage, "pending", req.RequestedBy, req.TimeoutAt)
	return err
}

func (r *PGGateRequestRepository) UpdateRequestStatus(ctx context.Context, id string, status string, approvedBy string, result string) error {
	_, err := r.db.ExecContext(ctx, `
		UPDATE gate_request SET status = $2, approved_by = $3, approved_at = NOW(), result = $4, updated_at = NOW()
		WHERE id = $1
	`, id, status, approvedBy, result)
	return err
}

func (r *PGGateRequestRepository) GetPendingRequests(ctx context.Context) ([]*port.DBGateRequest, error) {
	rows, err := r.db.QueryContext(ctx, `
		SELECT id, pipeline_id, stage, status, requested_by, timeout_at, created_at FROM gate_request
		WHERE status = 'pending' ORDER BY created_at ASC
	`)
	if err != nil {
		return nil, err
	}
	defer rows.Close()

	var list []*port.DBGateRequest
	for rows.Next() {
		var req port.DBGateRequest
		if err := rows.Scan(&req.ID, &req.PipelineID, &req.Stage, &req.Status, &req.RequestedBy, &req.TimeoutAt, &req.CreatedAt); err != nil {
			return nil, err
		}
		list = append(list, &req)
	}
	return list, nil
}

func (r *PGGateRequestRepository) GetActiveRequest(ctx context.Context, pipelineID, stage string) (*port.DBGateRequest, error) {
	var req port.DBGateRequest
	err := r.db.QueryRowContext(ctx, `
		SELECT id, pipeline_id, stage, status, requested_by, timeout_at, created_at FROM gate_request
		WHERE pipeline_id = $1 AND stage = $2 AND status = 'pending'
	`, pipelineID, stage).Scan(&req.ID, &req.PipelineID, &req.Stage, &req.Status, &req.RequestedBy, &req.TimeoutAt, &req.CreatedAt)
	if err == sql.ErrNoRows {
		return nil, nil
	}
	return &req, err
}
```

- [ ] **Step 1: 编写后台 Go 协程循环扫描超时**

在服务初始化时启动单独协程扫描超时的 `gate_request`。若检测到 `timeout_at` 已经早于 `NOW()` 且状态为 `pending` 的记录，则执行如下状态转换：

```go
func StartGateTimeoutChecker(db *sql.DB, qeManager *QueryEngineManager) {
    go func() {
        for {
            time.Sleep(10 * time.Second)
            // SQL 扫描
            rows, _ := db.Query(`UPDATE gate_request SET status = 'timeout', updated_at = NOW() WHERE status = 'pending' AND timeout_at < NOW() RETURNING pipeline_id`)
            for rows.Next() {
                var pid string
                rows.Scan(&pid)
                // 唤醒并取消对应的 QueryEngine 协程，防止死等
                qeManager.Abort(pid, "gate approval timeout (5 minutes limit)")
            }
            rows.Close()
        }
    }()
}
```
保证没有任意 Pipeline 在审批挂起时能无限期死锁系统。
