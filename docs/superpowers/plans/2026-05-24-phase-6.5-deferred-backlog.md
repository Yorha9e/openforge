
**SkillLoader** (参照 Spec §3.1):

```go
type SkillLoader struct {
    scanDirs   []string               // [global, team, project]
    configPath string                 // skill_config.yaml 路径 (per directory)
    watcher    *fsnotify.Watcher
    mu         sync.Mutex
    current    atomic.Value           // *skillSnapshot
    cache      sync.Map               // key: cacheKey → []Skill
    polling    *time.Ticker           // 30s fallback
}

func NewSkillLoader(scanDirs []string) (*SkillLoader, error)
func (sl *SkillLoader) Match(req MatchRequest) []Skill
func (sl *SkillLoader) MatchByName(name string) (*Skill, error)
func (sl *SkillLoader) Reload()
```

**Match 逻辑** (参照 Spec §4.1):
1. stage 精确过滤
2. 排除 deprecated/enabled=false/permission 不匹配
3. 计算 triggerPoints: file_patterns(+30) + user_intent(+20) + keywords(+15×命中率)
4. matchScore = triggerPoints / 50.0
5. finalScore = CurrentPriority × matchScore
6. 排序 (score → level → priority → version → createdAt → name)
7. 同名多版本取 CurrentPriority 最高者
8. top-K 截断 (maxSkillInject=5)
9. 缓存结果到 sync.Map

**边界条件** (参照 Spec §4.3): 无命中返回空列表、全部 deprecated 返回空+WARN、tokenBudget==0 返回 pending_user

**Reload**: mu.Lock → buildSnapshot → current.Store → 清空 cache。全量重建，失败保留旧 snapshot。

**Watch**: fsnotify Event (debounce 1s) → Reload()，30s polling ticker fallback

**降级** (参照 Spec §9): 启动失败 → emptySnapshot，Reload 失败 → 保留旧 snapshot，config.yaml 损坏 → 从 .md 重建

### Task 7: UnifiedPriorityEngine

**目标:** 每日 03:00 自动更新 Skill 优先级 + 弃用检测

**文件:**
```
新建:
  internal/agent/domain/priority_engine.go            ← UnifiedPriorityEngine (~200行)
  internal/agent/domain/priority_engine_test.go       ← Test
```

**算法** (参照 Spec §6):

```
CurrentPriority = BasePriority × VersionFactor × LearningFactor

VersionFactor:
  is_latest → 1.0
  !is_latest → e^(-0.1 × days_since_latest_published)，min 0.05

LearningFactor:
  usageCount < 10 → 1.0
  usageCount ≥ 10 → AcceptRate(7d) × 0.7 + sigmoid(usageCount) × 0.3
  范围 [0.1, 2.0]
```

**弃用**: CurrentPriority < 0.15 连续 3 天 → 标记 deprecated → 写入 skill_config.yaml → 人工确认。30d 不响应自动移除。

**配置** (参照 Spec §11):

```yaml
# config/skill_engine.yaml
skill:
  max_inject: 5
  avg_skill_tokens: 500
  priority:
    default_base: 70
    decay_mode: "exponential"
    decay_rate: 0.1
    max_decay_days: 60
    min_version_factor: 0.05
    window_days: 7
    accept_weight: 0.7
    usage_weight: 0.3
    deprecate_threshold: 0.15
    min_usage_for_learning: 10
  watch:
    debounce_ms: 1000
    poll_fallback_s: 30
```
