# Phase 8.4: 学习引擎深化与 A/B 实验数据闭环 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 落地知识版本快照、双实验组 A/B 分流器、以及事后复盘报告（Retrospective）机制，让 OpenForge 在每一次 Pipeline 执行完结后，由系统提炼出 lesson-learned 促使核心偏好（Preference）库在 A/B 科学检验中完成自演进。

**Architecture:** 
1. 编写快照(`knowledge_snapshot`)、回顾(`pipeline_retrospective`)与分流(`ab_experiment`、`ab_experiment_assignment`)的仓储逻辑。
2. 每一个新 Pipeline 在 `Create` 阶段通过 `A/B Experiment Assignment` 被路由随机赋予 Cohort A (默认策略权重) 或 Cohort B (激活实验组高权重策略)。
3. Pipeline 执行结束后，异步启动 `RetrospectiveService`，驱动 LLM 根据 `trajectory` 轨迹提炼归纳 lessons 并记录。
4. 按实验周期在统计显着性水平上，基于 CAR（代码接受率）自动决定推广、保留或丢弃 Cohort B 组实验新知识。

**Tech Stack:** Go, PostgreSQL, OpenAI/DeepSeek (LLM), 假设检验(AB/T-test)

- [ ] **Step 1: 新建并实现 `ExperimentStore` 适配器**

在 `internal/agent/adapter/pg_experiment_store.go` 中编写操作，将 A/B 实验关系插入并维护：

```go
package adapter

import (
	"context"
	"database/sql"
	"openforge/internal/agent/domain"
)

type PGExperimentStore struct {
	db *sql.DB
}

func NewPGExperimentStore(db *sql.DB) *PGExperimentStore {
	return &PGExperimentStore{db: db}
}

func (s *PGExperimentStore) AssignCohort(ctx context.Context, assignment *domain.ABAssignment) error {
	_, err := s.db.ExecContext(ctx, `
		INSERT INTO ab_experiment_assignment (experiment_id, pipeline_id, cohort)
		VALUES ($1, $2, $3)
	`, assignment.ExperimentID, assignment.PipelineID, assignment.Cohort)
	return err
}

func (s *PGExperimentStore) GetAssignment(ctx context.Context, pipelineID string) (*domain.ABAssignment, error) {
	var a domain.ABAssignment
	err := s.db.QueryRowContext(ctx, `
		SELECT experiment_id, pipeline_id, cohort FROM ab_experiment_assignment WHERE pipeline_id = $1
	`, pipelineID).Scan(&a.ExperimentID, &a.PipelineID, &a.Cohort)
	if err == sql.ErrNoRows {
		return nil, nil
	}
	return &a, err
}
```

```go
package adapter

import (
	"context"
	"database/sql"
	"github.com/lib/pq"
	"openforge/internal/agent/domain"
)

type PGRetrospectiveStore struct {
	db *sql.DB
}

func NewPGRetrospectiveStore(db *sql.DB) *PGRetrospectiveStore {
	return &PGRetrospectiveStore{db: db}
}

func (s *PGRetrospectiveStore) Save(ctx context.Context, r *domain.PipelineRetrospective) error {
	_, err := s.db.ExecContext(ctx, `
		INSERT INTO pipeline_retrospective (pipeline_id, project_id, duration_seconds, chat_rounds, total_tokens, lessons_learned)
		VALUES ($1, $2, $3, $4, $5, $6)
	`, r.PipelineID, r.ProjectID, r.DurationSeconds, r.ChatRounds, r.TotalTokens, pq.Array(r.LessonsLearned))
	return err
}
```

- [ ] **Step 1: 实现 Pipeline 完成后的自动分析触发器**

在 `learning_service.go` 中，当收到 Pipeline 完结的回调时，获取其对话轨迹，异步触发 LLM 复盘：

```go
func (s *LearningService) HandlePipelineCompleted(ctx context.Context, pipelineID string) {
	go func() {
		// 1. 获取轨迹
		t, _ := s.trajectoryStore.GetByPipeline(ctx, pipelineID)
		
		// 2. 调用 LLM
		lessons := s.llm.AnalyzeTrajectoryAndExtractLessons(t)
		
		// 3. 固化写入
		_ = s.retrospectiveStore.Save(ctx, &domain.PipelineRetrospective{
			PipelineID:     pipelineID,
			ProjectID:      t.ProjectID,
			LessonsLearned: lessons,
		})
	}()
}
```
