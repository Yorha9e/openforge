# OpenForge 路演大纲

> 时长：3-5 分钟 | 目标评委：技术专家 + 投资人

---

## Slide 1 · 封面（15秒）

**标题：** OpenForge — AI 驱动的全栈开发工作台

**副标题：** 从需求到部署，一切通过对话完成

**关键信息：**
- 一句话定位：让 PM 通过自然语言对话，完成需求→设计→编码→测试→部署全程
- 比赛：Agent 辅助全栈挑战赛 · 题一「AI 工程工具」

---

## Slide 2 · 痛点 + 解法（30秒）

**三个痛点：**
1. 需求到代码之间有巨大的沟通断层
2. 全栈开发涉及太多环节（需求/设计/实现/测试/部署/审计）
3. 现有 AI 编码工具只是"补全"，不是"工程"

**我们的解法：**
```
你说需求 → PM Agent 澄清拆解 → Pipeline 引擎调度 → Agent Swarm 执行 → 部署验证
```

**一句话：** 不只是写代码的 Copilot，而是管全流程的 Autopilot。

---

## Slide 3 · 三层正交架构（40秒）

```
┌─────────────────────────────────────────┐
│  C层 · 工作台 (React + Dockview + WS)    │  ← 开发者看到的多面板 IDE
├─────────────────────────────────────────┤
│  A层 · Pipeline 引擎 (Go · 状态机)       │  ← 9状态转换 · 审批门禁 · 部署
├─────────────────────────────────────────┤
│  B层 · Agent 集群 (Go+Node · CSP通信)    │  ← 多Agent协作 · 工具调用 · 自学习
├─────────────────────────────────────────┤
│  基础设施 (PG · Redis · Docker · Nginx)   │
└─────────────────────────────────────────┘
```

**亮点：** C/A/B 三层正交，每层独立演进，通过 Proto/gRPC/WebSocket 解耦

## Slide 4 · 核心能力矩阵（45秒）

| 模块 | 能力 | 技术含金量 |
|------|------|-----------|
| **对话式 PM** | NL→结构化需求→任务拆解 | Prompt 4层拼接 + 技能系统 |
| **Pipeline 状态机** | 9状态10动作 + 审批门禁 + 回溯 | 形式化状态转移矩阵 |
| **多Agent协作** | Spawn/Delegate/Broadcast | CSP Channel (Go原生) |
| **Pro Mode IDE** | Chat+Diff+文件树+终端+拓扑图 | Dockview 多面板布局 |
| **LLM 路由** | Anthropic/DeepSeek/OpenAI | 多Provider + Token计量 |
| **安全沙箱** | Docker隔离 + 5层防御 | LRU热池 + 命令注入防护 |
| **合规审计** | SHA256哈希链 + WORM | 月度分区 + 防篡改 |
| **渐进运维** | 3档配置 (minimal→standard→enterprise) | YAML切换，同一套代码 |

## Slide 5 · 协议与接口（20秒）

**40+ REST端点 + 12种WebSocket事件 + 6个gRPC服务**

```
REST API  ──→  项目管理 / Pipeline / 审批 / 成本 / 管理后台
WebSocket ──→  实时聊天流 / 工具代理 / 阶段变更 / Token预警
gRPC     ──→  Agent调度 / LLM路由 / 工具注册 / 学习引擎
```

**2个入口：** CLI（`of3 chat`）用于本地开发 · Web 工作台用于团队协作

---

## Slide 6 · 数据库设计亮点（20秒）

**22张表，核心设计原则：**

- UUID v7 主键 — 时间有序、全局唯一
- WORM 审计日志 — SHA256 哈希链，月度分区，法定留存
- 软删除 + 30天清理 — 数据可恢复
- 乐观锁（version字段）— 并发安全
- 会话分支 — 消息可编辑/可分支/可合并
